
[SO-NAME产生的原因](https://www.zhihu.com/tardis/zm/art/365882107?source_id=1003)

libaa.so.x.y.z
x 主版本号		不同x，版本不兼容
y 次版本号		不同y之间可以有接口新增
z 发布版本号    代表bugfix等

+ realname  即文件名
+ soname    libaa.so.x  
  作用于用户程序运行时的加载阶段，动态链接器会根据用户程序编译时记录的soname查找对应的共享库文件
+ linkname  libaa.so   
  是没有任何版本编号的文件名   
  用户程序编译阶段，链接器使用linkname来寻找对应的共享库（GCC中使用-l选项指定库，如-laaa，链接器就会去找libaaa.so），然后将共享库的soname记录在用户程序的动态链接信息中。

#### 查看SONAME

**使用带版本号的三方库时，如果出现链接和运行错误，可以关注一下SONAME**

```bash
readelf -d rpc.so | grep SONAME
```
