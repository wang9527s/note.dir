```undefined reference to UpdateRemoteDeviceInfo```无法解析的外部符号，老盆友了，经常会碰到的。  
一般就下面这四个原因

+ 没有指定库文件
+ 没有引入头文件
+ 函数没实现
+ 调用的地方写错了，没这个函数

```
$ find ./build/-name "lib*.a" -exec nm -C {} \;  | grep UpdateRemoteDeviceInfo
      U xx::xx::UpdateRemoteDeviceInfo()
```

+ 分析

这个问题是实际项目中遇到的，release版本编译没问题，debug(-g -O0)版本编译失败。

一开始以为是debug和release的宏不一致，现在天琴代码中出现了太多的宏，可能哪里写错了。

最后定位是：
没有定义AAA宏，实际上函数UpdateRemoteDeviceInfo不会被调用，release(-O3)优化了这块；debug(-O0)保留了UpdateRemoteDeviceInfo函数的调用，编译错误。

伪代码如下

{
#if !defined(AAA)
	...
	return;
#endif
	UpdateRemoteDeviceInfo()
}


吐槽：项目真大，宏又多，明明有实现，找了半天才找到原因。
UpdateRemoteDeviceInfo有声明和实现，但是整个network模块都被裁掉了，没有实现。然后加上刚刚的AAA宏，所以出现了这个奇怪的现象。
