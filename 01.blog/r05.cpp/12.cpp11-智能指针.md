## 1. 智能指针的引入

定义一个对象，其作用域为{}之间；当大括号结束时，程序调用类的析构函数。定义的是类指针，即p=new className();时。需要手动调用delete p;来释放空间。

但如果没有执行或者执行多次delete，会出现内存泄露甚至奔溃的情况。为了解决此问题，我们使用智能指正，用一个对象来管理野指针，使得在该对象构造时获得该指针管理权，析构时自动释放(delete)。

## 2. C++11中引入了新的智能指针

```shared_ptr```，基于**引用计数**的智能指针，会统计当前有多少个对象同时拥有该内部指针；当引用计数降为0时，自动释放。  
```weak_ptr```，基于引用计数的智能指针（shared_ptr）在面对循环引用的问题将无能为力，因此C++11还引入weak_ptr与之配套使用，**weak_ptr只引用，不计数，配合shared_ptr一起使用**。  
```unique_ptr```，遵循**独占语义的智能指针**，在任何时间点，资源智能唯一地被一个unique_ptr所占有，当其离开作用域时自动析构。资源所有权的转移只能通过std::move()而不能通过赋值。

### 2.1 shared_ptr 和 weak_ptr

**shared_ptr**

[推荐使用make_shared构造](https://www.jianshu.com/p/03eea8262c11)

[c++实现引用计数](https://blog.csdn.net/blqzj214817/article/details/121275016)。引用计数是线程安全的。但在多线程环境下，修改智能指针的指向，非线程安全（修改智能指针的内部指针和修改计数的操作有可能被打断，出现非想要的结果）。对智能指针指向的内容进行修改，那就更加不是线程安全了。

**weak_ptr**

weak_ptr看起来更像shared_ptr的附属品，它从shared_ptr衍生，但不会控制所指向对象的生命周期；**和shared_ptr共享一个引用计数**。

weak_ptr无法直接访问对象，需要通过weak_ptr.lock方法生成一个shared_ptr，再通过shared_ptr访问对象。

```cpp
#include <iostream>
#include <memory>

class AA {
public:
    int a = 200;
    AA(){ std::cout << __FUNCTION__ << std::endl;}
    ~AA(){ std::cout << __FUNCTION__ << std::endl;}
};

int main() {
    std::shared_ptr<AA> sp1 = std::make_shared<AA>();
    std::weak_ptr<AA> wp = sp1; // shared_ptr 构造 weak_ptr

    std::cout << "use_count after weak_ptr: " << sp1.use_count() << std::endl; // 1

    {
        auto sp2 = wp.lock(); // 升级 weak_ptr
        std::cout << "use_count after lock: " << sp1.use_count() << std::endl; // 2
        std::cout << sp2->a << std::endl;
    }

    std::cout << "use_count after sp2 destroyed: " << sp1.use_count() << std::endl; // 1

    // use_count == 0 后，释放T对应的空间，所以下次weak_ptr.lock获取到的 shared_ptr == nullptr
    sp1.reset();
    // 输出AA的析构

    std::cout << "use_count reset: " << sp1.use_count() << std::endl; // 0
    std::cout << std::string(sp1 == nullptr ? "sp1 is null" : "sp1 not null") << std::endl; // sp1 is null
    {
        auto sp3 = wp.lock();
        std::cout << std::string(sp3 == nullptr ? "sp3 is null" : "sp3 not null") << std::endl; // sp3 is null
    }
}
```

### 2.2 unique_ptr

&emsp;&emsp;由于unique_ptr对于内存的独占特性，当试图将一个 unique_ptr 赋值给另一个时。

- 如果源 unique_ptr 是个临时右值，编译器允许这么做: p2 = p1.release()
- 如果源 unique_ptr 将存在一段时间，编译器将禁止这么做，而只能支持右值引用的赋值：```unique_ptr<MyString> p2(move(p1))```

```cpp
#include <iostream>
#include <memory>

class AA {
public:
    int a = 200;
    AA(){ std::cout << __FUNCTION__ << std::endl;}
    ~AA(){ std::cout << __FUNCTION__ << std::endl;}
};

int main() {
    std::unique_ptr<AA> p0 = std::make_unique<AA>();
    // std::unique_ptr<AA> p1 = p0 // 禁止
    std::unique_ptr<AA> p1 = std::move(p0);
    AA* raw = p1.release(); // 放弃对T的占用，并且返回裸指针
    std::cout << raw->a << std::endl; // 200
}
```

```cpp
// Disable copy from lvalue.
unique_ptr(const unique_ptr&) = delete;               // 禁止拷贝构造函数（复制构造函数）
unique_ptr& operator=(const unique_ptr&) = delete;    // 禁止重载赋值运算符
```

### 2.3 shared_ptr的循环引用

```cpp
struct A {
    std::shared_ptr<B> bptr;
    ~A() {}
};

struct B {
    std::shared_ptr<A> aptr;
    ~B() {}
};

int main() {
    auto aaptr = std::make_shared<A>();
    auto bbptr = std::make_shared<B>();
    aaptr->bptr = bbptr;
    bbptr->aptr = aaptr;
    return 0;
}
```

上面代码，产生了循环引用，导致aptr和bptr的引用计数为2。离开作用域后，计数为1，内存不会被释放。

将类内部的一个shared_ptr改为weak_ptr；需要的时候，使用weak_ptr.lock升级为shared_ptr，然后使用。

### 3. QSharedPointer

Qt智能指针的源码 （看的头晕）

```cpp
namespace {
    struct ExternalRefCountData
    {
        typedef void (*DestroyerFn)(ExternalRefCountData *);
        QBasicAtomicInt weakref;
        QBasicAtomicInt strongref;
        DestroyerFn destroyer;
    }
    /*
    1. 使用 QSharedPointer::create构造，T是构造在栈上的
    ExternalRefCountData 的子类中有 data成员，create的时候，shared_ptr->value指向data。

    2. 使用指针构造，其他的子类实现
    */
    void destroy() { destroyer(this); }
}
class QSharedPointer {
    ExternalRefCountData *d;
    T *value;

    // 计数 为0 删除T对应的空间
    // weak_count 为0 删除自己
    static void deref(Data *dd) noexcept
    {
        if (!dd) return;
        if (!dd->strongref.deref()) {
            dd->destroy();
        }
        if (!dd->weakref.deref())
            delete dd;
    }
}
```
